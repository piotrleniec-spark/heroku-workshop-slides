Chapter 2

"Super" Feature

Backstory

One of the developers (people call him "Wizard Developer") finished a very important feature. The biggest file has 1435 lines of code, there are no tests and the code is "peculiar" in the best case. On the top of it all, everything is in a single commit named "change". Deadline is near so we have no choice but to merge it and "test" it on production.
Set dyno count to 1

Let's test that feature!

  1. Visit /login
  2. Sign in using spree@example.com/spree123 credentials
  3. Visit /admin
  4. Click Configuration -> General Settings
Set dyno count to 2 and repeat the process

Conclusion

Running at least 2 web dynos for any mission-critical app increases the probability that the app will remain available during a catastrophic event.
